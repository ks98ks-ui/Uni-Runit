# Runit

## 프로젝트 소개
Runit은 짧고 직관적인 조작으로 즐길 수 있는 2D 러닝 액션 게임입니다.
플레이어는 계속 앞으로 진행하는 흐름 속에서 점프를 활용해 발판을 넘고 장애물을 피하면서 최대한 오래 살아남아 높은 점수를 기록하는 것을 목표로 합니다.

이 프로젝트는 단순히 캐릭터를 움직이는 데서 끝나는 것이 아니라, 반복 플레이가 자연스럽게 이어질 수 있도록 게임 흐름과 구조를 함께 설계하는 데 의도를 두고 제작했습니다.

## 제작 의도
Runit은 누구나 바로 이해할 수 있는 조작 방식과 빠른 게임 템포를 가진 러닝 게임을 만드는 것을 목표로 했습니다.
마우스 클릭만으로 점프를 제어할 수 있게 구성해 접근성을 높였고, 배경 이동, 발판 재배치, 장애물 배치, 점수 증가, 게임 오버 처리까지 한 흐름 안에서 자연스럽게 이어지도록 구성했습니다.

또한 단순한 플레이 경험 뒤에서 게임을 안정적으로 관리할 수 있도록, 전역 게임 상태를 통제하는 구조와 반복 오브젝트를 효율적으로 다루기 위한 구조를 함께 고려했습니다.

## 핵심 기능
- 클릭 입력으로 점프와 점프 높이 조절이 가능한 플레이어 조작
- 발판 착지 판정과 2단 점프 처리
- 발판을 밟을 때 점수가 증가하는 점수 시스템
- 장애물 충돌 시 게임 오버 처리
- 무한히 이어지는 느낌을 주는 배경 반복 이동
- 랜덤 장애물 활성화로 매번 조금씩 달라지는 플레이 흐름

## 적용한 구조와 의도

### 1. 싱글턴 패턴
이 프로젝트에서는 `GameManager`를 싱글턴으로 구성했습니다.

싱글턴을 사용한 이유는 게임 전체에서 공통으로 사용하는 상태를 한 곳에서 관리하기 위해서입니다.
점수, 게임 오버 여부, UI 표시, 플레이어 사망 처리 같은 요소는 여러 오브젝트가 함께 참조해야 하기 때문에, 매번 참조를 따로 연결하기보다 하나의 공용 매니저를 두는 방식이 더 효율적이라고 판단했습니다.

이 구조를 통해 얻고자 한 효과는 다음과 같습니다.

- 게임 상태를 한 곳에서 일관성 있게 관리할 수 있음
- 다른 스크립트에서 `GameManager.instance`로 쉽게 접근 가능
- 점수 처리나 게임 오버 처리 흐름이 단순해짐
- 프로젝트 규모가 커져도 관리 포인트를 명확하게 유지할 수 있음

### 2. 오브젝트 풀링
이 프로젝트에서는 발판과 같은 반복 오브젝트를 효율적으로 관리하기 위해 오브젝트 풀링 구조를 고려했습니다.

러닝 게임은 발판이나 장애물이 반복적으로 등장하고 사라지기 때문에, 매번 새로 생성하고 삭제하는 방식은 비효율적일 수 있습니다.
그래서 미리 오브젝트를 만들어 두고 필요할 때 재사용하는 방식으로 관리하려는 의도를 두었습니다.

이 구조를 넣으려는 이유는 다음과 같습니다.

- 반복 생성과 삭제를 줄여 성능 부담을 낮추기 위해
- 오브젝트 재사용으로 게임 흐름을 더 안정적으로 유지하기 위해
- 발판 배치 시스템을 확장하기 쉽게 만들기 위해

현재 프로젝트에서는 발판 재배치를 위한 구조가 준비되어 있으며, 이후 오브젝트 풀링 로직을 구체화하면 러닝 게임 특성에 더 잘 맞는 최적화가 가능하도록 설계 방향을 잡고 있습니다.

## 주요 시스템 설명

### 플레이어
플레이어는 클릭 입력으로 점프하며, 상황에 따라 2단 점프가 가능합니다.
버튼을 누른 뒤 떼는 타이밍에 따라 점프 높이가 달라지도록 구성해 단순한 조작 안에서도 플레이 감각을 살리고자 했습니다.

### 발판과 장애물
발판은 단순한 바닥 역할만 하는 것이 아니라 점수 획득의 기준이 됩니다.
또한 발판 위 장애물이 랜덤하게 활성화되도록 하여, 반복 플레이에서도 완전히 같은 패턴이 되지 않도록 구성했습니다.

### 배경과 스크롤
배경과 오브젝트는 왼쪽으로 이동하며, 배경은 일정 지점에 도달하면 다시 오른쪽으로 재배치됩니다.
이를 통해 맵이 계속 이어지는 듯한 러닝 게임의 흐름을 표현했습니다.

## 기대한 플레이 경험
Runit은 복잡한 시스템보다도 “짧게 시작해서 바로 몰입할 수 있는 러닝 게임”의 감각을 만드는 데 집중한 프로젝트입니다.
단순한 조작, 반복되는 도전, 점수 갱신의 재미를 중심으로 설계했으며, 그 안에서 게임 매니저 구조와 오브젝트 재사용 구조 같은 기본적인 게임 개발 설계 방식도 함께 적용해보는 것을 목표로 했습니다.
